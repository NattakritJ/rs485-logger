### Stale REQUIREMENTS.md Checkboxes (4 items)

### Orphaned REQ-ID

`REL-01` is referenced in `ROADMAP.md:82` but has no definition in `REQUIREMENTS.md`. The 12 concrete findings all fixed; only the traceability document needs updating.

### Missing VERIFICATION.md Files (5 phases)

Phases 1, 2, 4, 5, and 6 were executed without `/gsd-verify-phase`. Requirements for these phases are confirmed satisfied via SUMMARY.md frontmatter and integration checker, but no formal verification report exists.

### CFG-02 Parity Gap

`SerialConfig` only contains `port` and `baud_rate`. The `parity` field (mentioned in CFG-02) is absent. The PZEM-016 uses 8N1 by default and tokio_serial defaults to `None` parity (effectively 8N1), so this works in practice. However, the requirement is technically unsatisfied for "configurable parity". 

**Recommended fix for v1.1:** Add `parity: Option<String>` to `SerialConfig` with default `"N"` and map to tokio_serial Parity enum in `ModbusPoller::new()`.

### Documentation Mismatch (RISK-1)

`deploy/99-rs485.rules` uses `ch341` driver while comment and README say `cp210x`. Fix one or the other before shipping.
