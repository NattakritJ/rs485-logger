<feature>
  <name>to_line_protocol</name>
  <files>src/influx.rs, src/main.rs</files>
  <behavior>
    InfluxDB 3 line protocol format (STOR-01, STOR-03):
      `{measurement} voltage={v},current={c},power={p},energy={e},frequency={f},power_factor={pf} {ts_ns}`

    Rules:
    - Measurement name = `reading.device_name`
    - Timestamp = `reading.timestamp_secs * 1_000_000_000` (nanoseconds)
    - All numeric values MUST be formatted as floats, never integers.
      Use Rust format `"{:.4}"` or similar — InfluxDB 3 locks field type on first write;
      an integer literal (e.g. `power=0`) would create an integer field permanently.
    - Field order: voltage, current, power, energy, frequency, power_factor
    - No tags (measurement name alone identifies the device per STOR-01)
    - Single space between field set and timestamp (line protocol spec)

    Test cases:
    - Basic: PowerReading { device_name: "solar_panel", voltage: 230.1, current: 1.234,
        power: 285.2, energy: 10240.0, frequency: 50.0, power_factor: 0.95,
        timestamp_secs: 1700000000 }
      → "solar_panel voltage=230.1000,current=1.2340,power=285.2000,energy=10240.0000,frequency=50.0000,power_factor=0.9500 1700000000000000000"

    - Zero power: PowerReading { ..., power: 0.0, ... }
      → power field contains "0." or ".0" or "0.0000" — NEVER bare "0"
      This is the STOR-03 critical test: zero must be float-typed

    - Measurement name with underscore: device_name = "grid_meter"
      → "grid_meter voltage=..."  (underscore is valid in measurement names, no escaping needed)

    - Long device name: device_name = "building_a_floor_2_panel"
      → measurement name in output = "building_a_floor_2_panel" verbatim
  </behavior>
  <implementation>
    Create `src/influx.rs`:
    ```rust
    use crate::types::PowerReading;

    /// Convert a PowerReading to InfluxDB 3 line protocol format.
    /// Format: measurement field_key=field_value,... timestamp_ns
    /// All numeric fields are floats (STOR-03: prevents InfluxDB 3 type conflicts).
    pub fn to_line_protocol(reading: &PowerReading) -> String {
        let ts_ns = reading.timestamp_secs * 1_000_000_000_i64;
        format!(
            "{} voltage={:.4},current={:.4},power={:.4},energy={:.4},frequency={:.4},power_factor={:.4} {}",
            reading.device_name,
            reading.voltage,
            reading.current,
            reading.power,
            reading.energy,
            reading.frequency,
            reading.power_factor,
            ts_ns,
        )
    }
    ```
