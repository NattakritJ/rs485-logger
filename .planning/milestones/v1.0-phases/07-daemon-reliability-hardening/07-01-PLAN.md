# Plan 07-01: InfluxDB Client Hardening + Git Hygiene + Log Rotation

## Findings Addressed

- **CRIT-01**: InfluxDB HTTP requests have no timeout
- **CRIT-03**: API token committed to git
- **HIGH-01**: No timeout on error response body read
- **HIGH-03**: Database name not URL-encoded
- **MED-01**: Log file grows unbounded
- **MED-02**: far_future() duration unnecessarily large

## Tasks

### T1: Add connect + request timeout to reqwest::Client (CRIT-01 + HIGH-01)

**File:** `src/influx.rs:35-36`

Replace `reqwest::Client::new()` with:
```rust
let client = reqwest::Client::builder()
    .connect_timeout(std::time::Duration::from_secs(5))
    .timeout(std::time::Duration::from_secs(10))
    .build()
    .context("Failed to build HTTP client")?;
```

Change `InfluxWriter::new()` to return `anyhow::Result<Self>` and update callers.

The `.timeout()` applies to the entire request lifecycle including response body read, which fixes both CRIT-01 and HIGH-01.

### T2: Validate database name at config time (HIGH-03)

**File:** `src/config.rs:53-78`

Add to `validate_config()`:
```rust
anyhow::ensure!(
    cfg.influxdb.database.chars().all(|c| c.is_alphanumeric() || c == '_' || c == '-'),
    "influxdb.database '{}' contains invalid characters (use only alphanumeric, underscore, dash)",
    cfg.influxdb.database
);
```

This is simpler and more robust than URL-encoding at runtime — database names should be simple identifiers.

### T3: Add config.toml to .gitignore + create config.toml.example (CRIT-03)

- Add `config.toml` to `.gitignore`
- Copy current `config.toml` to `config.toml.example` with placeholder token

### T4: Switch log appender to rolling::daily (MED-01)

**File:** `src/main.rs:102`

Replace `rolling::never(dir, filename)` with `rolling::daily(dir, filename)`.

Note: `rolling::daily` appends a date suffix to the filename, so the file will be named like `rs485.log.2026-04-03`. This is the standard tracing-appender behavior.

### T5: Reduce far_future() to 10 years (MED-02)

**File:** `src/main.rs:43`

Change `365 * 24 * 3600 * 100` to `365 * 24 * 3600 * 10`.

### T6: Update tests + verify build

- Add test for database name validation
- Update `InfluxWriter::new()` callsites (now returns Result)
- Run `cargo test` and `cargo clippy`
