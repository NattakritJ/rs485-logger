# Plan 07-02: Config Validation Hardening

## Findings Addressed

- **HIGH-02**: Device name not sanitized for InfluxDB line protocol
- **MED-05**: Energy reset timezone/time validated lazily at startup
- **LOW-02**: Epoch-0 timestamp if system clock is wrong at boot

## Tasks

### T1: Validate device names at config load (HIGH-02)

**File:** `src/config.rs` — `validate_config()`

Add validation that device names contain only alphanumeric characters and underscores:
```rust
for device in &cfg.devices {
    anyhow::ensure!(
        !device.name.is_empty(),
        "device at address {} has empty name",
        device.address
    );
    anyhow::ensure!(
        device.name.chars().all(|c| c.is_alphanumeric() || c == '_'),
        "device name '{}' contains invalid characters — use only alphanumeric and underscore (spaces, commas, newlines break InfluxDB line protocol)",
        device.name
    );
}
```

### T2: Validate energy_reset timezone and time at config load (MED-05)

**File:** `src/config.rs` — `validate_config()`

When `energy_reset` is present and enabled, validate timezone and time eagerly:
```rust
if let Some(ref er) = cfg.energy_reset {
    if er.enabled {
        er.timezone.parse::<chrono_tz::Tz>()
            .map_err(|_| anyhow::anyhow!("Unknown timezone '{}' in energy_reset", er.timezone))?;
        chrono::NaiveTime::parse_from_str(&er.time, "%H:%M")
            .with_context(|| format!("Invalid time '{}' in energy_reset — expected HH:MM", er.time))?;
    }
}
```

This turns a runtime silent failure into a startup fatal error. Add `chrono` as a direct dependency if not already (it may be transitive via `chrono-tz`).

### T3: Warn on suspicious system clock (LOW-02)

**File:** `src/types.rs`

After computing `timestamp_secs`, add:
```rust
if timestamp_secs < 1_704_067_200 { // 2024-01-01 00:00:00 UTC
    tracing::warn!(
        timestamp_secs,
        "System clock appears incorrect (before 2024-01-01) — data may have wrong timestamps"
    );
}
```

Use a `std::sync::Once` or `AtomicBool` to log this warning only once (not every poll).

### T4: Add unit tests

- Test device name with spaces is rejected

- Test device name with newline is rejected
- Test empty device name is rejected
- Test valid device name passes (alphanumeric + underscore)
- Test invalid timezone is rejected when energy_reset.enabled = true
