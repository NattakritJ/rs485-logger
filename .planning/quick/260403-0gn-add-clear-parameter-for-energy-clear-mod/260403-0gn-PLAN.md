```rust
let mut clear_mode = false;

    }

```

After config is loaded and tracing is initialised (so the operator sees structured
log output), insert an early-exit branch **before** the `ModbusPoller::new` call and
before the poll loop is set up. The branch should:

1. Log `tracing::info!("--clear mode: sending energy reset to all devices")`.
2. Open the Modbus port via `ModbusPoller::new(&cfg.serial)?`.
3. Iterate `cfg.devices` sequentially (same pattern as the reset arm in the poll loop):
   ```rust
   for device in &cfg.devices {
       tracing::info!(device = %device.name, "Energy reset sending command");
       match poller.reset_energy(device).await {
           Ok(()) => tracing::info!(device = %device.name, "Energy reset OK"),
           Err(e) => tracing::warn!(device = %device.name, error = %e,
                                     "Energy reset failed, skipping"),
       }
   }
   ```
4. Log `tracing::info!("--clear mode: done")`.
5. `return Ok(());` — exits before the InfluxWriter, ticker, or loop are created.

The `InfluxWriter` is NOT constructed in clear mode — no `cfg.influxdb` access needed.
The existing normal-mode code below (InfluxWriter, loop, select!) is **unchanged**.

No new dependencies. No changes to `poller.rs`, `config.rs`, or any other file.
