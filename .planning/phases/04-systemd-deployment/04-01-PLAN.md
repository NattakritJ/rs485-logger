Create the `deploy/` directory and write two files.

**`deploy/rs485-logger.service`** — full systemd unit:
```ini
[Unit]
Description=RS485 Logger — PZEM-016 power meter polling daemon
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=rs485logger
Group=rs485logger
SupplementaryGroups=dialout
ExecStart=/usr/local/bin/rs485-logger --config /etc/rs485-logger/config.toml
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal
SyslogIdentifier=rs485-logger

# Hardening
NoNewPrivileges=yes
ProtectSystem=strict
ProtectHome=yes
ReadWritePaths=/var/log/rs485-logger
PrivateTmp=yes

[Install]
WantedBy=multi-user.target
```

Key decisions:
- `SupplementaryGroups=dialout` gives serial port access without running as root (per D-07 from STACK.md: RPi serial port group is `dialout`)
- `After=network-online.target` ensures InfluxDB HTTP writes work on boot (handles slow Pi DHCP)
- `Restart=always` + `RestartSec=5` satisfies OPS-04 exactly
- `ProtectSystem=strict` + `ReadWritePaths=/var/log/rs485-logger` matches the log file path pattern from Phase 3 (tracing-appender writes here)
- `User=rs485logger` — dedicated non-root service user (created by install.sh)

**`deploy/99-rs485.rules`** — udev rule for stable device naming:
```
# USB-RS485 adapter → /dev/ttyRS485 stable symlink
# Rule matches by subsystem + driver; add ATTRS{idVendor}=="XXXX",ATTRS{idProduct}=="XXXX" if multiple adapters present
SUBSYSTEM=="tty", SUBSYSTEMS=="usb", DRIVERS=="cp210x", SYMLINK+="ttyRS485", MODE="0660", GROUP="dialout"
```

Add a comment block at the top of 99-rs485.rules explaining:
1. How to find vendor/product IDs: `udevadm info -a -n /dev/ttyUSBx | grep -E 'idVendor|idProduct'`
2. How to test without reboot: `sudo udevadm control --reload && sudo udevadm trigger`
3. Common USB-RS485 chip drivers: `cp210x` (SiLabs CP2102), `ch341` (CH340/CH341), `ftdi_sio` (FTDI)

The `cp210x` driver covers the most common cheap USB-RS485 adapters. Operator adds `ATTRS{idVendor}` refinement only if they have multiple USB serial devices.

<task type="auto">
  <name>Task 2: install.sh deployment script</name>
  <files>deploy/install.sh</files>

Create `deploy/install.sh` — a bash script that performs a complete one-shot install on Raspberry Pi OS. Must be idempotent (safe to re-run).

```bash
#!/usr/bin/env bash
set -euo pipefail

BINARY_SRC="${1:-./rs485-logger}"   # first arg or default to cwd binary
CONFIG_DIR=/etc/rs485-logger
LOG_DIR=/var/log/rs485-logger
SERVICE_USER=rs485logger
BINARY_DEST=/usr/local/bin/rs485-logger
SERVICE_FILE=/etc/systemd/system/rs485-logger.service
UDEV_RULE=/etc/udev/rules.d/99-rs485.rules

echo "=== rs485-logger install ==="

# 1. Create service user (idempotent)
if ! id "$SERVICE_USER" &>/dev/null; then
    useradd --system --no-create-home --shell /usr/sbin/nologin "$SERVICE_USER"
    echo "Created system user: $SERVICE_USER"
fi

# 2. Add service user to dialout group
usermod -aG dialout "$SERVICE_USER"

# 3. Install binary
install -m 755 "$BINARY_SRC" "$BINARY_DEST"
echo "Installed binary: $BINARY_DEST"

# 4. Create config directory (operator must populate config.toml)
mkdir -p "$CONFIG_DIR"
chmod 750 "$CONFIG_DIR"
chown "root:$SERVICE_USER" "$CONFIG_DIR"

# 5. Create log directory
mkdir -p "$LOG_DIR"
chown "$SERVICE_USER:$SERVICE_USER" "$LOG_DIR"

# 6. Install systemd service
install -m 644 "$(dirname "$0")/rs485-logger.service" "$SERVICE_FILE"
systemctl daemon-reload
systemctl enable rs485-logger.service
echo "Enabled systemd service"

# 7. Install udev rule
install -m 644 "$(dirname "$0")/99-rs485.rules" "$UDEV_RULE"
udevadm control --reload-rules
udevadm trigger
echo "Installed udev rule"

echo ""
echo "=== Install complete ==="
echo "Next steps:"
echo "  1. Copy your config:  cp config.toml $CONFIG_DIR/config.toml"
echo "  2. Secure it:         chmod 600 $CONFIG_DIR/config.toml && chown $SERVICE_USER:$SERVICE_USER $CONFIG_DIR/config.toml"
echo "  3. Start daemon:      systemctl start rs485-logger"
echo "  4. Check status:      systemctl status rs485-logger"
echo "  5. Watch logs:        journalctl -u rs485-logger -f"
```

Make the script executable: `chmod +x deploy/install.sh`

Notes embedded in script:
- Accepts the compiled binary path as `$1` — operator passes the cross-compiled release binary
- `ProtectSystem=strict` in the service unit requires `ReadWritePaths=/var/log/rs485-logger` — the script creates this with correct ownership
- `set -euo pipefail` — exits immediately on any error, no silent failures
