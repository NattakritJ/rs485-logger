<task type="auto">
  <name>Task 1: Create Cargo.toml with all dependencies</name>
  <files>Cargo.toml</files>
  <action>
Create `Cargo.toml` at the project root. Use the exact versions locked in research:

```toml
[package]
name = "rs485-logger"
version = "0.1.0"
edition = "2021"

[[bin]]
name = "rs485-logger"
path = "src/main.rs"

[dependencies]
tokio = { version = "1.50.0", features = ["rt", "time", "signal", "io-util", "macros"] }
tokio-modbus = { version = "0.17.0", default-features = false, features = ["rtu"] }
tokio-serial = "5.4.5"
reqwest = { version = "0.13.2", default-features = false, features = ["rustls-tls"] }
serde = { version = "1.0.228", features = ["derive"] }
toml = "1.1.1"
tracing = "0.1.44"
tracing-subscriber = { version = "0.3.23", features = ["env-filter"] }
tracing-appender = "0.2.4"
anyhow = "1.0.102"

[profile.release]
opt-level = 3
strip = true
```

Key decisions:
- `tokio` features: `rt` (single-thread runtime), `time` (interval), `signal` (SIGTERM), `io-util`, `macros` (`#[tokio::main]`). Do NOT use `full` — adds unnecessary deps.
- `tokio-modbus`: `default-features = false, features = ["rtu"]` — disables TCP feature, keeps binary lean.
- `reqwest`: `rustls-tls` (not `native-tls`) — avoids OpenSSL, essential for cross-compilation to ARM targets in Phase 4.
- No `chrono` dependency — use `std::time::SystemTime` for timestamps (returns `i64` epoch seconds) to keep dependency count minimal in Phase 1.
  </action>

    <automated>cargo build 2>&1 | head -5 || true</automated>
  </verify>
  <done>Cargo.toml exists; `cargo fetch` or `cargo build` downloads all dependencies without version conflicts.</done>

Use `#[tokio::main(flavor = "current_thread")]` — NOT the default multi-threaded flavor. This matches the RS485 single-bus sequential polling architecture documented in ARCHITECTURE.md (Pattern 1). The single-threaded runtime eliminates `Send` bound requirements on serial port handles.

Do NOT add any `use` imports or `todo!()` macros yet — keep stubs minimal so `cargo build` passes with zero warnings.
